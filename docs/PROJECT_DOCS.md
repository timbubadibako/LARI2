# LARI2: Project Feature Documentation

## 1. Authentication
- **Login/Register:** Secure access via JWT-based authentication with encrypted password storage. Users are identified by their unique Agent ID, allowing faction/guild assignment.

## 2. Dashboard (The Grid)
- **Main Map:** The central tactical interface. It displays the global map, current territory status (captured areas in faction colors), and real-time location.
- **Faction Dominion:** Visual breakdown of faction influence across the map.

## 3. Run Activity
- **Live Tracking:** Real-time geospatial tracking with high-precision telemetry (Pace, Distance, Time).
- **Pocket Mode:** Optimized power-saving tracking state for continuous background logging without device thermal throttling.
- **Elastic Trails:** Custom snap-to-road algorithms for accurate path matching in urban environments.

## 4. Post-Activity
- **Mission Summary:** Visual feedback upon completing a run. Displays total area acquired, territory captured, and overall performance statistics.
- **Territory Conquest:** Automatic calculation and filling of captured regions on the map.

## 5. History
- **Dossier Archive:** Comprehensive log of all past missions, categorizing runs into 'Standard Activity' (Routes) and 'Area Acquisition' (Territories).
- **Mission Preview:** Each log includes a static map preview of the route taken, allowing users to review their tactical footprint.

## 6. Social Room
- **Community Feed:** Real-time feed showing recent successful acquisitions and global activities.
- **Run Together (Coming Soon):** Feature to host or join sessions via shared PINs for group operations.
- **Leaderboard:** Global ranking based on secured territory (SQM).

## 7. Settings
- **Configuration:** Personalize faction colors, profile data, and system configurations.
